# /component-review — 컴포넌트 설계 판별 스킬

11전략 렌즈로 컴포넌트 설계 냄새를 판별한다.
판별만 한다. 수정 여부와 방법은 사용자가 직접 결정한다.

**추측하지 말고 실제 파일을 읽어 확인한다.** 근거 없는 발견을 만들지 않는다.

---

## 절차

1. 경로를 지정하면 해당 경로의 .tsx 파일을 읽는다. 지정하지 않으면 src/ 안의 .tsx 파일을 모두 읽는다.
2. `CLAUDE.md`를 읽어 프로젝트 규칙을 확인한다.
3. 아래 11전략을 하나씩 적용해 판별하고 결과를 표로 보고한다.

---

## 11전략 체크리스트

### 경계

**① 변화의 경계**
질문: 이 컴포넌트가 바뀌는 이유가 몇 가지인가?
냄새 신호 : 바뀌는 이유가 2가지 이상이다.
해당 없음 기준 : 바뀌는 이유가 하나이다.

**② 구현 vs 조합**
질문 : 이미 있는 컴포넌트를 재사용하지 않고 비슷한 걸 새로 만들고 있지는 않은가?
냄새 신호 : 비슷한 UI나 로직이 두 군데 이상 중복되어 있다.
해당 없임 기준 : 중복 없이 각자 역할이 다르다

**③ God Component**
질문: 이 컴포넌트가 너무 많은 걸 알고 있나?
냄새 신호: `useState`가 5개 이상이다
해당 없음 기준: `useState`가 5개 미만이다

**④ 성급한 추상화**
질문: 지금 추상화할 필요가 있나?
냄새 신호: 한 곳에서만 쓰이는데 분리되어 있고, 분리 이유를 한 문장으로 설명할 수 없다
해당 없음 기준: 분리 이유가 명확하다 (자체 state, 재사용, 복잡도)

---

### 계약

**⑤ props는 적게, 이름은 역할대로**
질문: props가 너무 많지 않나? 이름이 뭘 하는 건지 바로 보이나?
냄새 신호: props가 5개를 넘거나, 역할을 알 수 없는 이름(`data`·`flag`·`value`)이 props 이름으로 쓰인다. 밖에서 값을 써야 하는데 props가 없어 제어할 수 없다
해당 없음 기준: props가 5개 이하이고 이름만 봐도 역할이 명확하다

**⑥ boolean 폭발 → enum**
질문: boolean props가 여러 개인데 사실 하나의 상태 아닌가?
냄새 신호: boolean props가 여러 개인데 동시에 두 개가 true가 될 수 없다
해당 없음 기준: 각 boolean이 독립적으로 true/false가 될 수 있다

**⑦ 파생 상태**
질문: 이 state, 사실 계산하면 나오는 값 아닌가?
냄새 신호: `useState`에 담긴 값이 다른 state나 props로 계산할 수 있다
해당 없음 기준: state가 외부에서 오거나 계산으로 나오지 않는 독립적인 값이다

**⑧ 확장은 위임으로**
질문: 새 케이스가 생길 때 이 컴포넌트 내부를 고쳐야 하나?
냄새 신호: if문·삼항연산자가 타입·케이스별로 줄줄이 있어서 새 케이스가 생기면 내부를 고쳐야 하는 구조다
해당 없음 기준: 케이스별 if문·삼항연산자가 없다

---

### 합성

**⑨ Context 전에 composition**
질문: ⑪에서 drilling이 발견됐을 때, Context 쓰기 전에 합성으로 먼저 풀 수 있나?
냄새 신호: drilling이 있는데 합성으로 풀 수 있는 구조임에도 Context를 바로 도입하려 한다
해당 없음 기준: drilling이 없거나, 중간 컴포넌트가 연관된 state를 가져서 합성으로 풀기 어렵다

**⑩ children vs slot**
질문: 같은 컴포넌트를 2곳 이상에서 다른 UI로 쓰고 있나?
냄새 신호: 같은 컴포넌트인데 쓰는 곳마다 UI가 달라야 해서 props로 분기가 생기고 있다
해당 없음 기준: 한 곳에서만 쓰이거나, 쓰는 곳마다 UI가 동일하다

**⑪ Drilling vs Context**
질문: props가 중간 컴포넌트를 그냥 통과만 하나?
냄새 신호: 중간 컴포넌트가 props를 쓰지도 않으면서 아래로 넘기기만 한다
해당 없음 기준: 모든 컴포넌트가 받은 props를 직접 사용한다

## 판별 정책

- 코드를 자동으로 고치지 않는다. 판별만 한다.
- "해당 없음"으로 판단한 전략도 반드시 1줄 근거와 함께 적는다.
- 고치지 않기로 한 냄새도 이유와 함께 "보류 판단"에 적는다.
- 냄새가 아닌데 "냄새다"라고 하지 않는다. 근거 없는 발견을 만들지 않는다.
